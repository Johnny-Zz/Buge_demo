## 💡 产品背景与愿景

在当下碎片化信息爆炸的校园环境中，大学生面临着极高的“时间统筹认知负荷”：群聊通知繁杂易漏、课程表与临时任务频繁冲突、个人习惯难以量化追踪。

**“不鸽”** 致力于将大语言模型（LLM）的泛化理解能力与 GTD（Getting Things Done）时间管理哲学深度融合，打造一个“懂上下文、懂你的习惯、懂统筹避坑”的 AI 校园日程管家。

## ✨ 核心业务亮点 (Core Features)

### 1. 📥 全域信息无感摄入 (Smart Inbox)
- **QQ 群聊“一键召唤”**：智能解析群聊上下文，自动提取任务的 `[动作]`、`[时间戳]`、`[空间坐标]` 与凭证附件，支持查重防漏。
- **自然语言模糊排程**：支持“我要去健身”、“明晚复习高数”等极简口语指令，AI 自动结合全局语境与用户习惯精准排期。
- **动态习惯快捷键**：将用户的长期习惯提炼为高频快捷按钮，实现零摩擦的任务录入。

### 2. ⚙️ 柔性任务处理枢纽 (GTD Engine)
- **双轨收纳机制**：明确任务一键「加入日程」，待定任务一键「暂存」至专属收集栏。
- **防呆反悔设计**：所有任务状态（入库、暂存）均支持无缝切换与一键撤回。
- **入库前预编辑 (Inline Edit)**：支持在智能解析卡片上直接二次修正时间、地点与优先级 (P0-P2)。

### 3. 📅 智能运筹与时空避险 (Conflict Radar)
- **教务底座强约束**：无缝对接模拟课表，作为全局时间轴的底层不可侵犯资产。
- **插空排布算法**：大模型结合底座空闲时间，进行见缝插针式的任务分配。
- **双向冲突预警**：实时碰撞检测，当任务与课表、任务与任务发生时空重叠时，触发高亮红框警报拦截。

## 🛠️ 纯前端架构与技术栈

本 Demo 专注于高保真产品交互体验与前端状态流转：
- **框架**：Next.js (App Router) + React
- **样式**：Tailwind CSS + Radix UI (暗黑科技风设计语言)
- **状态管理**：Zustand (单点真实数据源，实现全域视图数据的实时联动与双向绑定)
- **原型生成**：基于 v0.dev AI 辅助构建
